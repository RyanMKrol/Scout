# T016 — Live radio provider (CoreTelephony)

## Overview

The on-device implementation of `RadioInfoProviding`: reads the active data service's radio access
technology via CoreTelephony, maps it to Scout's `RadioGeneration`, and re-reads on change
notifications. The mapping is a pure, fully unit-tested function; the provider is a thin wrapper.
(The simulator has no baseband — CoreTelephony returns nil/empty there — so the simulator keeps the
simulated provider.)

## Do

1. Create `Sources/Live/RadioTechnologyMapping.swift` — pure:

```swift
enum RadioTechnologyMapping {
    static func generation(fromRadioAccessTechnology rat: String?) -> RadioGeneration
}
```

   Exact mapping (these are the complete set of `CTRadioAccessTechnology*` constants; compare against
   the CONSTANTS, not hardcoded strings):
   - `CTRadioAccessTechnologyNR`, `CTRadioAccessTechnologyNRNSA` → `.fiveG`
   - `CTRadioAccessTechnologyLTE` → `.lte`
   - `CTRadioAccessTechnologyWCDMA`, `HSDPA`, `HSUPA`, `CDMAEVDORev0`, `CDMAEVDORevA`,
     `CDMAEVDORevB`, `eHRPD` → `.threeG`
   - `CTRadioAccessTechnologyGPRS`, `CTRadioAccessTechnologyEdge` (note: "Edge", lowercase "dge"),
     `CTRadioAccessTechnologyCDMA1x` → `.twoG`
   - `nil` or ANY unrecognized/future value → `.unknown` (forward-compatible default — never crash).

2. Create `Sources/Live/CoreTelephonyRadioProvider.swift`:

```swift
final class CoreTelephonyRadioProvider: RadioInfoProviding { init() }
```

   - Own one `CTTelephonyNetworkInfo`. Current value = look up
     `info.serviceCurrentRadioAccessTechnology?[info.dataServiceIdentifier ?? ""]` (guard the
     identifier; nil → `.unknown`).
   - `generations()`: yield the current generation immediately, then observe
     `NotificationCenter.default` for `.CTServiceRadioAccessTechnologyDidChange` (the notification's
     `object` is the changed service-identifier String; `userInfo` is nil — you RE-READ the
     dictionary rather than reading the notification). Notifications arrive on a NON-main
     CoreTelephony queue — the provider must not touch UI; just yield the re-read value into the
     stream. Remove the observer in the stream's `onTermination`.
   - No entitlement, permission prompt, or Info.plist key is needed for these APIs.

3. Wire the composition root (`Sources/App/AppEnvironment.swift`): on DEVICE builds
   (`#if !targetEnvironment(simulator)`) use `CoreTelephonyRadioProvider()` for the radio provider —
   UNLESS a `-ScoutScenario` launch argument is explicitly present (then keep the simulated one, all
   platforms). Simulator always keeps `SimulatedRadioProvider`. Sampler/path wiring unchanged.

## Done when

- A test file (e.g. `Tests/RadioTechnologyMappingTests.swift`) asserts the mapping for ALL 13
  constants by name (import CoreTelephony in tests), plus `nil → .unknown` and
  `"CTRadioAccessTechnologyFutureThing" → .unknown`.
- Simulator behavior is unchanged (screenshot check not required; the provider must simply never be
  constructed there outside the device branch).
- Standard DoD run green. Real-device behavior is verified later by the human device-test gate.
