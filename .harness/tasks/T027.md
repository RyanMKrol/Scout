# T027 — Simulated providers (scripted async streams)

## Overview

The simulated implementations of the T006 seams that Scout uses in the iOS Simulator (which has no
cellular radio), in UI tests, and in the harness's screenshot checks. Thin async wrappers around the
already-tested `ScenarioWalk` math — the timing loop is the only new logic.

## Do

Create `Sources/Domain/SimulatedProviders.swift`:

```swift
final class SimulatedSampler: ThroughputSampling { init(scenario: SimulationScenario) }
final class SimulatedRadioProvider: RadioInfoProviding { init(scenario: SimulationScenario) }
final class SimulatedPathMonitor: CellularPathMonitoring { init(scenario: SimulationScenario) }
```

- `SimulatedSampler.samples()`: every 250 ms emit ONE download sample then ONE upload sample, values
  driven by `ScenarioWalk` (`nextDownloadMbps` from the scenario's `initialDownloadMbps`, then
  `uploadMbps(forDownload:)`; use `SystemRandomNumberGenerator`). Fabricate probe shape:
  `transferDuration = .milliseconds(180)`, `byteCount = Int(mbps × 1_000_000 × 0.18 / 8)`,
  `endedAt = ContinuousClock().now` at emission (real now is fine here — this is the live-timing
  wrapper; the pure math was already tested with a seeded rng). Sleep via
  `try? await ContinuousClock().sleep(for: .milliseconds(250))` inside a `Task` owned by the
  stream; the stream's `onTermination` MUST cancel that task. Scenario `.none` → yield nothing
  (stream stays open, no samples).
- `SimulatedRadioProvider.generations()`: immediately yield — great: `.fiveG`; usable: `.fiveG`;
  poor: `.lte`; live: `.fiveG`; none: `.unknown` — then keep the stream open.
- `SimulatedPathMonitor.availability()`: immediately yield `false` for `.none`, `true` otherwise;
  then keep the stream open.

## Done when

- A test (e.g. `Tests/SimulatedProvidersTests.swift`) asserts, with bounded waits (never unbounded):
  - collecting the first 4 samples from `SimulatedSampler(scenario: .great).samples()` yields
    alternating directions starting with `.download`, positive byte counts, and 180 ms durations;
  - `SimulatedRadioProvider(scenario: .none)` yields `.unknown` as its first element;
    `SimulatedPathMonitor(scenario: .none)` yields `false`, and `.great` yields `true`;
  - cancelling the Task consuming `samples()` stops emission (the stream's termination path runs —
    e.g. no new samples arrive after cancellation within a short grace window).
- Tests are hermetic (no networking) and pass in the standard DoD run.
