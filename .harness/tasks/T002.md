# T002 — README truth-up: align the concept doc with the settled v1 design

## Overview

The root `README.md` predates the design handoff and several product decisions; this task rewrites its
concept/UX sections so the README accurately describes what v1 Scout actually is. The design handoff at
`design/design_handoff_scout/README.md` is the SOURCE OF TRUTH for the design — where the old README
disagrees with it, the handoff wins.

## Do

Edit `README.md` (only) so it reflects these settled decisions. Keep the existing honest
"what iOS allows / throughput-not-bars" framing and overall tone; rewrite the stale parts:

1. **Dual-direction, capped instrument.** Scout measures BOTH download and upload cellular throughput.
   Download is the hero metric; upload is a secondary reading. Readings are deliberately CAPPED:
   download at 10 Mbps (displayed "10+"), upload at 5 Mbps (displayed "5+"). Rationale: the product
   question is "is this spot good enough to load a video / send messages?", not "how fast exactly" —
   capping lets the app probe with small paced transfers (~256 KB) so real data use stays roughly
   10–40 MB per minute, matching the consent copy.
2. **Quality bands (download-driven):** Great ≥ 6 Mbps, Usable ≥ 2 Mbps, Poor < 2 Mbps. Color + label
   always together. Upload never affects the verdict color.
3. **No Start/Stop buttons, no trend graph, no history.** The app measures automatically whenever the
   measuring screen is foreground (per the handoff: "It opens and it is already measuring"). Remove
   the "rolling trend graph" and "Clear Start/Stop" bullets from Core UX and the roadmap.
4. **Screens:** Splash → first-run consent (once) → Measuring (dual-arc radar dial). Fallback states:
   No-cellular (airplane mode / no SIM / Wi-Fi-only), and an idle "sweeping is off" state when consent
   was declined ("Not now").
5. **P0 requirement — record it prominently:** the moment the app leaves the foreground (backgrounded,
   app switcher, screen lock), ALL data transfer stops immediately. No background measurement of any kind.
6. **Data source:** v1 downloads from / uploads to Cloudflare's public speed endpoints
   (`https://speed.cloudflare.com/__down?bytes=N` and `/__up`), pinned in one config constant;
   a self-hosted endpoint remains a later option.
7. **Design source of truth:** state that `design/design_handoff_scout/` (README + HTML prototype)
   is the authoritative design and the app recreates it natively in SwiftUI.
8. Update the "Roadmap (rough)" checklist to reflect the actual backlog phases (foundation math →
   simulated measurement engine → screens → live cellular providers → accessibility/UI tests →
   icon/App-Store readiness). Do NOT touch the "Build status" table or `.harness/` files.
