# T004 — Domain model: quality bands, radio generation, capped display formatting, dial scale (pure)

## Overview

The pure domain vocabulary and math every other part of Scout composes on: quality bands, radio
generation, the capped "10+"/"5+" display formatting, MB formatting, and the dial's log-scale arc
fractions. All pure, fully unit-tested — no I/O, no Date(), no SwiftUI.

## Do

Create these types under `Sources/Domain/` (one or more files), with EXACTLY these public shapes —
later tasks bind to these signatures verbatim:

```swift
enum RadioGeneration: String, Sendable, Equatable {
    case fiveG = "5G"
    case lte = "LTE"
    case threeG = "3G"
    case twoG = "2G"
    case unknown = ""          // simulator / Wi-Fi-only / airplane / unregistered — UI hides it
}

enum SignalQuality: Sendable, Equatable {
    case great, usable, poor
    init(downloadMbps: Double)   // ≥ 6.0 → great; ≥ 2.0 → usable; else poor
    static let greatThresholdMbps = 6.0
    static let usableThresholdMbps = 2.0
}

enum ScoutMeter {
    static let downloadCapMbps = 10.0
    static let uploadCapMbps = 5.0
    static func downloadDisplay(_ mbps: Double) -> String
    static func uploadDisplay(_ mbps: Double) -> String
    static func megabytesDisplay(bytes: Int64) -> String
    static func downloadArcFraction(_ mbps: Double) -> Double
    static func uploadArcFraction(_ mbps: Double) -> Double
}
```

Exact behaviors (Scout is a deliberately CAPPED instrument — see README):

- `downloadDisplay`: round to 1 decimal first (`(mbps * 10).rounded() / 10`); if the rounded value
  ≥ 10.0 return `"10+"`, else format with exactly one decimal (`"7.4"`, `"0.0"`). Negative input → `"0.0"`.
- `uploadDisplay`: same rule with cap 5.0 → `"5+"`.
- `megabytesDisplay(bytes:)`: MB = bytes / 1_000_000 (decimal megabytes, Double). If MB < 10 → one
  decimal (`"3.2 MB"`), else rounded integer (`"14 MB"`). 0 → `"0.0 MB"`. Always suffix `" MB"`.
- `downloadArcFraction`: `clamp(log10(min(mbps, 10) + 1) / log10(11), 0.04, 1.0)` — the handoff's
  log-scale arc formula retuned to the 10 Mbps cap (handoff allows threshold/scale tuning). Inputs
  ≤ 0 → 0.04.
- `uploadArcFraction`: `clamp(log10(min(mbps, 5) + 1) / log10(6), 0.04, 1.0)`.
- Quality boundaries are inclusive: 6.0 → great, 2.0 → usable, 1.999… → poor.
