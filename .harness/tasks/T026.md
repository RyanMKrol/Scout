# T026 — 🔒 Human gate: design-fidelity review against the prototype

## Overview

Final taste gate: the owner compares the built app side-by-side with the design house's interactive
prototype and signs off v1's fidelity. Screenshot checks during the build verified each screen in
isolation; this is the holistic pass.

## Do

(Human, interactively:)

1. Open `design/design_handoff_scout/Scout.dc.html` in a browser (use the **Scenario** control to
   force Great / Usable / Poor) and run the app beside it (`./build_run.sh` with the matching
   `-ScoutScenario` args, or a device).
2. Compare all four surfaces: splash (2a), measuring (2b — all three scenario colors), no-cellular
   (2c), consent (3a). Check: colors, spacing, type sizes/weights, the dual-arc dial geometry, motion
   (sweep speed ~3.2–3.4 s, pulse, arc smoothness at 4 Hz — no strobing), copy character-for-character.
3. Two deviations are INTENTIONAL and owner-approved — verify they read well rather than flagging
   them: (a) the capped instrument — hero pegs at "10+", upload at "5+", bands Great ≥ 6 / Usable
   ≥ 2 (the handoff's "tune with real-world data" allowance); (b) the consent-declined idle state
   ("Sweeping is off" + Start button), which the handoff doesn't spec.
4. Record every unintentional deviation found (screenshot + note) as ideas/backlog follow-ups.

## Done when

The comparison is done, deviations are captured as follow-ups, and the owner accepts v1's fidelity —
then `.harness/scripts/mark-done.sh T026`.
