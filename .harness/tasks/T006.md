# T006 — Measurement seams (frozen protocols) + scenario-walk math (pure)

## Overview

The protocol contract every measurement source implements, plus the pure random-walk math the
simulated providers will use. Live cellular/radio I/O can never run in CI, so these seams are what
make the whole app testable — a frozen contract later tasks bind to verbatim. This task is types +
pure functions ONLY; the async simulated providers that wrap them are the NEXT task (T027).

## Do

1. Create `Sources/Domain/MeasurementSeams.swift` with EXACTLY these shapes:

```swift
enum TransferDirection: Sendable, Equatable {
    case download, upload
}

struct ThroughputSample: Sendable, Equatable {
    let direction: TransferDirection
    let byteCount: Int                       // payload bytes moved (HTTP headers excluded)
    let transferDuration: Duration           // active transfer time for those bytes
    let endedAt: ContinuousClock.Instant
}

protocol ThroughputSampling: Sendable {
    /// Single-subscriber stream of probe results. Finishing/cancelling the consuming task MUST
    /// tear down all underlying transfer activity (P0: no data moves once the consumer stops).
    func samples() -> AsyncStream<ThroughputSample>
}

protocol RadioInfoProviding: Sendable {
    /// Yields the current generation immediately on subscription, then again on every change.
    func generations() -> AsyncStream<RadioGeneration>
}

protocol CellularPathMonitoring: Sendable {
    /// Yields current cellular availability immediately on subscription, then on every change.
    func availability() -> AsyncStream<Bool>
}
```

2. Create `Sources/Domain/ScenarioWalk.swift` — pure, seeded-RNG-driven simulation math (adapted
   from the design prototype's simulation in `design/design_handoff_scout/Scout.dc.html`, rescaled
   to Scout's capped 0–10 Mbps instrument):

```swift
enum SimulationScenario: String, Sendable, Equatable {
    case live = "Live", great = "Great", usable = "Usable", poor = "Poor", none = "None"
}

enum ScenarioWalk {
    static func initialDownloadMbps(_ scenario: SimulationScenario) -> Double
    static func nextDownloadMbps(previous: Double, scenario: SimulationScenario,
                                 using rng: inout some RandomNumberGenerator) -> Double
    static func uploadMbps(forDownload down: Double,
                           using rng: inout some RandomNumberGenerator) -> Double
}
```

   Exact math (rand01 = uniform Double in 0..<1 drawn from the rng):
   - Bands (download): great `[6.5, 9.8]`, usable `[2.4, 5.5]`, poor `[0.3, 1.4]`.
   - Initial values: great 8.15, usable 3.95, poor 0.85, live 4.0, none 0.
   - Banded scenarios: `next = prev + (center − prev) × 0.28 + (rand01 − 0.5) × (hi − lo) × 0.4`,
     then clamp to `[lo × 0.7, min(hi × 1.12, 10.0)]`.
   - `.live`: `next = prev + (rand01 − 0.5) × 1.3`; with probability 0.06 add `(rand01 − 0.4) × 4.3`;
     clamp to `[0.2, 9.9]`.
   - `.none`: return 0.
   - Upload: `down × 0.45 + (rand01 − 0.5) × 0.6`, clamped to `[0.1, 5.0]`.

## Done when

- A test file (e.g. `Tests/ScenarioWalkTests.swift`) with a seeded RNG (a tiny SplitMix64
  `RandomNumberGenerator` defined in the test target) asserts:
  - for each banded scenario, iterating `nextDownloadMbps` 200× from the initial value never leaves
    the clamp range, and all `.great` values stay ≥ 4.55 (6.5 × 0.7);
  - `.live` values stay within [0.2, 9.9] over 200 iterations;
  - `uploadMbps` stays within [0.1, 5.0], and over 50 draws the mean upload for `down = 8` exceeds
    the mean upload for `down = 1`.
