# T009 — App shell: RootView, composition root, launch-argument controls, foreground-only lifecycle

## Overview

Wire the app together: the `@main` entry composes providers (simulated everywhere for now — live ones
swap in via later tasks), RootView routes splash/consent/measuring via AppRouter with placeholder stub
screens (real screens replace them in later tasks), scenePhase enforces the P0 "background ⇒ all
transfer stops immediately" rule, and launch arguments make every state reachable for screenshots and
UI tests.

## Do

1. **Composition root** — `Sources/App/AppEnvironment.swift`:

```swift
@MainActor
enum AppEnvironment {
    /// Reads launch-argument-backed defaults and builds the app's providers.
    static func makeSession() -> SweepSession
    static func scenario() -> SimulationScenario   // parses UserDefaults "ScoutScenario"
}
```

   - `scenario()`: `UserDefaults.standard.string(forKey: "ScoutScenario")` parsed via
     `SimulationScenario(rawValue:)`, defaulting to `.live` (unknown strings → `.live`).
     iOS auto-populates UserDefaults from launch arguments (`-ScoutScenario Great`), so this is the
     whole launch-arg mechanism — document that in a comment.
   - `makeSession()`: `SweepSession(sampler: SimulatedSampler(scenario:), radio:
     SimulatedRadioProvider(scenario:), path: SimulatedPathMonitor(scenario:))`. Add a clearly marked
     seam comment: on-device LIVE providers replace these in later tasks (T016/T017/T019) — keep the
     construction of each provider on its own line so those tasks swap one line each.

2. **Root routing** — `Sources/App/RootView.swift` + update the `@main` App in `Sources/`:
   - `@AppStorage("firstRunConsentGiven") private var consentGiven = false` (launch arg
     `-firstRunConsentGiven NO`/`YES` overrides it automatically via the argument domain).
   - Create `AppRouter(consentGiven: consentGiven, holdSplash: UserDefaults.standard.bool(forKey:
     "ScoutHoldSplash"))` once (`@State`).
   - ZStack: content for `router.phase` (`.splash` → SplashStubView, `.consent` → ConsentStubView,
     `.measuring` → MeasuringStubView); splash overlays with `.zIndex(1)` and `.transition(.opacity)`.
     After a 1 s `.task` sleep, `withAnimation(.easeOut(duration: 0.4)) { router.splashFinished() }`.
   - Wire consent actions: onStart → `router.startSweeping()`; onNotNow → `router.declineConsent()`.
     Mirror `router.consentGiven` back into the `@AppStorage` var whenever it changes (`.onChange`).
   - Own the session: `@State private var session = AppEnvironment.makeSession()`, injected with
     `.environment(session)`.
   - **Foreground-only measurement (P0):** observe `@Environment(\.scenePhase)`; on `.active`, call
     `session.start()` ONLY IF `router.phase == .measuring && router.consentGiven`; on `.inactive` and
     `.background`, call `session.stop()` immediately — comment it as the P0 rule. Also start the
     session when the router first ENTERS measuring with consent (grant moment).
   - If UserDefaults bool `"ScoutForceIdle"` is set: initialize the router already at `.measuring`
     with `consentGiven = false` and skip the splash — add an `initialPhase` convenience to the
     RootView layer (do NOT change AppRouter's frozen init; e.g. call `splashFinished()` immediately
     and pass holdSplash false, with consentGiven forced false).
3. **Stub screens** — `Sources/Screens/SplashStubView.swift`, `ConsentStubView.swift`,
   `MeasuringStubView.swift`. Deliberately minimal (real screens land in later tasks), but on
   `ScoutTheme.background` full-screen (`.ignoresSafeArea()`) with these accessibility identifiers so
   UI tests written later hold:
   - splash: text "Scout" — id `splash.wordmark`
   - consent: title text + button "Start sweeping" (id `consent.start`) + button "Not now"
     (id `consent.notNow`)
   - measuring: shows `ScoutMeter.downloadDisplay(session.downloadMbps)` (id `measuring.hero`);
     when `!session.cellularAvailable` OR `!router.consentGiven`, instead show text "PAUSED"
     (id `paused.title`) and, if consent not given, a "Start sweeping" button (id `paused.startButton`)
     that calls the same start action.
4. **Delete the T001 "Hello, Scout" placeholder** content from ContentView / entry point as needed —
   the app now launches into RootView.
5. **`build_run.sh` launch-args passthrough** (this file is scope-exempt): change the launch line to
   `xcrun simctl launch "$SIM" "$BUNDLE_ID" ${SCOUT_LAUNCH_ARGS:-}` (unquoted expansion, so
   `SCOUT_LAUNCH_ARGS='-ScoutScenario Great -firstRunConsentGiven YES' ./build_run.sh` forces states).

## Done when

- Unit tests (e.g. `Tests/AppEnvironmentTests.swift`) cover `AppEnvironment.scenario()` parsing:
  each raw value maps to its case, missing/garbage → `.live` (set/remove the default in the test and
  clean up after — never leave test state in standard defaults).
- `./build_run.sh` screenshot shows the stub flow running (dark background, consent stub on first
  launch); `SCOUT_LAUNCH_ARGS="-firstRunConsentGiven YES -ScoutScenario Great" ./build_run.sh` shows
  the measuring stub with a nonzero, changing reading; `-ScoutScenario None` shows "PAUSED".
- scenePhase handling is present exactly as specced (P0 comment included), and the standard DoD run
  is green.
