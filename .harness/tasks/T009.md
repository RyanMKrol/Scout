# T009 — Composition root: AppEnvironment (providers + launch-argument scenario parsing)

## Overview

The single place Scout decides which providers feed the session: simulated everywhere for now (live
ones swap in via later tasks), with the scenario chosen by a launch-argument-backed UserDefaults key.
Pure construction + parsing — no UI in this task (RootView and screens come next).

## Do

Create `Sources/App/AppEnvironment.swift`:

```swift
@MainActor
enum AppEnvironment {
    /// Builds the session from the current launch-argument scenario.
    static func makeSession() -> SweepSession
    /// Parses UserDefaults "ScoutScenario" (auto-populated by a `-ScoutScenario <value>` launch
    /// argument — iOS puts dash-prefixed launch arguments into the UserDefaults argument domain).
    static func scenario() -> SimulationScenario
}
```

- `scenario()`: `UserDefaults.standard.string(forKey: "ScoutScenario")` parsed via
  `SimulationScenario(rawValue:)`; missing key or unknown string → `.live`.
- `makeSession()`: `SweepSession(sampler: SimulatedSampler(scenario: s), radio:
  SimulatedRadioProvider(scenario: s), path: SimulatedPathMonitor(scenario: s))`. Keep each
  provider's construction on its own line with a clearly marked seam comment: on-device LIVE
  providers (CoreTelephony / NWPathMonitor / NetworkConnection) replace these lines in later tasks
  (T016/T017/T019); the simulator ALWAYS keeps the simulated ones.

## Done when

- A test file (e.g. `Tests/AppEnvironmentTests.swift`) covers `scenario()` parsing: each raw value
  ("Live"/"Great"/"Usable"/"Poor"/"None") maps to its case; missing key and a garbage string both →
  `.live`. Tests set the default via `UserDefaults.standard.set(...)` and MUST remove it in
  teardown — never leave test state in standard defaults.
- `makeSession()` compiles returning a `SweepSession` wired to the three simulated providers.
- Standard DoD run green.
