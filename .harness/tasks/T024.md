# T024 — Privacy manifest (PrivacyInfo.xcprivacy)

## Overview

App Store submissions require a privacy manifest declaring required-reason API usage. Scout's only
flagged API is UserDefaults (via `@AppStorage` for the consent flag and the launch-argument scenario
plumbing); it tracks nothing and collects nothing.

## Do

1. Create `Sources/PrivacyInfo.xcprivacy` (Apple's XML plist format) declaring:
   - `NSPrivacyTracking`: `false`
   - `NSPrivacyTrackingDomains`: `[]`
   - `NSPrivacyCollectedDataTypes`: `[]`
   - `NSPrivacyAccessedAPITypes`: one entry —
     `NSPrivacyAccessedAPICategoryUserDefaults` with reasons `["CA92.1"]` (app accesses its own
     UserDefaults).
2. Ensure XcodeGen bundles it as a RESOURCE of the Scout target: files under `sources: [Sources]`
   are included by type, and `.xcprivacy` lands as a resource — verify rather than assume: after
   `xcodegen generate` and a simulator build (e.g. via `./build_run.sh`), check the built product
   contains it: `ls build/Debug-iphonesimulator/Scout.app/PrivacyInfo.xcprivacy`. If XcodeGen did
   NOT include it, add an explicit `resources`/`buildPhase: resources` entry for it in `project.yml`
   (smallest change that works).

## Done when

- `Sources/PrivacyInfo.xcprivacy` exists with exactly the four declarations above (valid plist XML —
  `plutil -lint Sources/PrivacyInfo.xcprivacy` passes).
- The built `Scout.app` bundle contains `PrivacyInfo.xcprivacy` at its root (verified, output noted
  in the worklog).
- Standard DoD run green.
