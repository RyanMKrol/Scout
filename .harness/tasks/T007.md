# T007 — SweepSession: the observable measurement session engine

## Overview

The central state machine the UI observes: consumes injected sampler/radio/path streams and publishes
capped download/upload Mbps, quality, generation, per-direction session byte counters, and cellular
availability. Also owns the P0 rule: `stop()` must immediately tear down all data flow.

## Do

Create `Sources/Domain/SweepSession.swift` with EXACTLY this public shape:

```swift
@MainActor @Observable
final class SweepSession {
    init(sampler: any ThroughputSampling,
         radio: any RadioInfoProviding,
         path: any CellularPathMonitoring)

    private(set) var downloadMbps: Double        // capped at ScoutMeter.downloadCapMbps
    private(set) var uploadMbps: Double          // capped at ScoutMeter.uploadCapMbps
    private(set) var quality: SignalQuality      // derived from downloadMbps (download only)
    private(set) var generation: RadioGeneration
    private(set) var sessionDownloadBytes: Int64
    private(set) var sessionUploadBytes: Int64
    private(set) var cellularAvailable: Bool     // starts true
    private(set) var isMeasuring: Bool

    func start()
    func stop()
}
```

Behavior:

- `start()` (idempotent): spawns consuming `Task`s for the three streams; `isMeasuring = true`.
  - Each arriving `ThroughputSample` goes into one of two `ThroughputWindow(window: .seconds(2))`
    (one per direction). After recording, recompute: `downloadMbps = min(windowValue ?? 0, 10)`,
    `uploadMbps = min(upWindowValue ?? 0, 5)`, `quality = SignalQuality(downloadMbps:)`, and add
    `byteCount` to the matching session byte counter (counters never reset while the app runs).
  - Radio stream → `generation`.
  - Path stream → `cellularAvailable`. On `false`: cancel the SAMPLER consuming task (transfers stop),
    zero `downloadMbps`/`uploadMbps` (quality follows), keep counters. On `true` while `isMeasuring`:
    re-subscribe to `sampler.samples()` with fresh windows.
- `stop()`: synchronously cancel ALL consuming tasks, `isMeasuring = false`. **P0 (non-negotiable):
  cancelling the sampler consumption is what tears down the transfer — the AsyncStream's
  `onTermination` must fire promptly.** Readings keep their last values (UI shows paused states).
- No `Date()`, no `Timer`, no owned clock: published values update on sample arrival (samples already
  arrive at the design cadence). Windows compare only the instants carried by samples — pass
  `sample.endedAt` as `now` when reading the window after a record.
- Keep ALL cellular/radio knowledge behind the injected protocols — this type must compile with zero
  imports beyond Foundation/Observation.

## Done when

- A test file (e.g. `Tests/SweepSessionTests.swift`) using hand-rolled fakes (a fake
  `ThroughputSampling` whose `AsyncStream` continuation the test drives, and which records whether
  `onTermination` fired) asserts at least:
  - feeding a download sample of 250_000 B / 0.2 s → `downloadMbps == 10.0` (capped exactly),
    `quality == .great`, `sessionDownloadBytes == 250_000`;
  - upload samples land in `uploadMbps` (capped at 5) and `sessionUploadBytes`, never in the
    download properties;
  - byte counters accumulate across multiple samples, split by direction;
  - `stop()` → fake sampler's stream terminated (onTermination observed within a bounded wait)
    and `isMeasuring == false`  ← the P0 test;
  - path fake yielding `false` → `cellularAvailable == false`, readings drop to 0, and the sampler
    stream is terminated; yielding `true` again → sampler re-subscribed (fake sees a second
    `samples()` call).
- Tests are hermetic (no networking, no live radio) and pass in the standard DoD run.
