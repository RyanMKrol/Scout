# T021 — XCUITest target + screen smoke tests (runs in CI)

## Overview

Add a UI-test target that launches the real app against scripted scenarios and asserts each screen's
key elements by accessibility identifier. These run in CI as part of the standard `xcodebuild test`
(owner-approved trade-off: a few extra CI minutes for screen-level regression protection).

## Do

1. **`project.yml`**: add a `ScoutUITests` target — `type: bundle.ui-testing`, `platform: iOS`,
   `sources: [UITests]`, `dependencies: [target: Scout]`, settings
   `PRODUCT_BUNDLE_IDENTIFIER: com.ryankrol.scout.uitests`, `GENERATE_INFOPLIST_FILE: YES`,
   `SWIFT_DEFAULT_ACTOR_ISOLATION: nonisolated`. Add `ScoutUITests` to the `Scout` scheme's `test`
   targets (alongside `ScoutTests`) and to `build` with `[test]`.
2. **`UITests/ScoutUITests.swift`** (XCTest / XCUITest). House rules (repo convention — restated here
   because they are binding): query ONLY by accessibilityIdentifier, NEVER display copy; every
   existence check uses a bounded wait (`waitForExistence(timeout: 15)`); NEVER assert exact live
   values (readings are random-walk driven) — assert element existence/hittability only. Each test
   sets `app.launchArguments` explicitly:
   - `testFirstRunConsentFlow`: args `["-firstRunConsentGiven", "NO"]` → `consent.start` and
     `consent.notNow` appear (splash passes by itself); tap `consent.start` → `measuring.hero`
     appears.
   - `testMeasuringScreenElements`: args `["-firstRunConsentGiven", "YES", "-ScoutScenario",
     "Great"]` → `measuring.hero`, `measuring.upload`, `measuring.quality`, `measuring.dataDown`,
     `measuring.dataUp` all appear.
   - `testNoCellularState`: args `["-firstRunConsentGiven", "YES", "-ScoutScenario", "None"]` →
     `paused.title` appears; `measuring.quality` does NOT exist after the wait.
   - `testDeclineThenStartFromIdle`: args `["-firstRunConsentGiven", "NO"]` → tap `consent.notNow` →
     `paused.startButton` appears; tap it → `measuring.hero` appears.
3. Confirm the full DoD command (which CI mirrors) now executes the UI tests too, and everything
   passes locally against the dedicated Scout-Sim (ensured by `tools/loop_sim.sh`; never a generic
   model name — see root CLAUDE.md).

## Done when

- `xcodegen generate` succeeds with the new target; the four tests above exist verbatim in intent
  (names may vary) and PASS in the standard DoD run (`xcodebuild test` on the Scout scheme now runs
  unit + UI tests).
- No test queries display copy or asserts a numeric reading value; all waits are bounded.
- CI (same command) is green on the branch.
