# T035 — 🔒 Device signing setup (Apple Developer team in project.yml)

## Overview

Running Scout on a physical iPhone (required by the T025 on-device verification gate) needs code
signing, which needs your Apple Developer team. The `.xcodeproj` is regenerated by XcodeGen, so the
team must be recorded in `project.yml` — settings clicked into Xcode's UI are lost on the next
`xcodegen generate`.

## Do

(Human, interactively:)

1. Have an Apple Developer account (free personal team works for direct device installs; paid is
   needed later for TestFlight — see T036).
2. Find your team id: Xcode → Settings → Accounts → your Apple ID → team; or
   `security find-certificate -c "Apple Development" -p | openssl x509 -noout -subject` (the OU).
3. In `project.yml`, add to the Scout target's `settings.base`:
   `DEVELOPMENT_TEAM: <your team id>` and `CODE_SIGN_STYLE: Automatic`. Run `xcodegen generate`.
4. Plug in an iPhone, select it as the run destination in Xcode, build and run Scout once. Accept
   the developer-trust prompt on the device (Settings → General → VPN & Device Management) if asked.
5. Commit the `project.yml` change (a team id is not a secret). Confirm CI stays green — simulator
   builds still run with `CODE_SIGNING_ALLOWED=NO`, so CI must be unaffected.

## Done when

Scout builds, installs, and launches on a physical iPhone from Xcode; `project.yml` carries the
team; CI is still green. Then `.harness/scripts/mark-done.sh T035`.
