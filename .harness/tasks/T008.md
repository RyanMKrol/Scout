# T008 — AppRouter: the app-phase state machine (pure)

## Overview

A small observable state machine deciding which top-level screen shows: splash → consent (first run)
→ measuring, including the "Not now" declined path (measuring in an idle state, consent not granted).
Pure — no timers, no UI, no persistence; the shell (next task) supplies timing and storage.

## Do

Create `Sources/App/AppRouter.swift` with EXACTLY this shape:

```swift
@MainActor @Observable
final class AppRouter {
    enum Phase: Equatable { case splash, consent, measuring }

    init(consentGiven: Bool, holdSplash: Bool = false)

    private(set) var phase: Phase          // starts .splash
    private(set) var consentGiven: Bool

    func splashFinished()
    func startSweeping()
    func declineConsent()
}
```

Rules:

- `phase` starts `.splash` regardless of consent.
- `splashFinished()`: if `holdSplash` is true, do nothing (stay on `.splash` — used by screenshot
  tooling). Else `phase = consentGiven ? .measuring : .consent`. Calling it when not on `.splash`
  does nothing.
- `startSweeping()`: sets `consentGiven = true` and `phase = .measuring`. Valid from `.consent` AND
  from `.measuring` (the idle "Start sweeping" button when consent was previously declined).
- `declineConsent()` ("Not now"): `phase = .measuring`, `consentGiven` stays `false` — the measuring
  screen renders its idle variant and NO data transfer may start. Only valid from `.consent`
  (no-op otherwise).
- The router never persists anything and never reads UserDefaults — the caller passes initial state in
  and observes `consentGiven` to persist changes.

## Done when

- A test file (e.g. `Tests/AppRouterTests.swift`) covers every transition:
  - fresh install (consentGiven false): splash → splashFinished → consent → startSweeping →
    measuring with consentGiven true;
  - returning user (consentGiven true): splash → splashFinished → measuring;
  - declined: consent → declineConsent → measuring with consentGiven false; then startSweeping →
    consentGiven true (idle-state grant);
  - holdSplash: splashFinished is a no-op, phase stays .splash;
  - out-of-phase calls (splashFinished twice, declineConsent from measuring) change nothing.
- Tests pass in the standard DoD run.
