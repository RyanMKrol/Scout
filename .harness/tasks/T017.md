# T017 — Live cellular path monitor (NWPathMonitor)

## Overview

The on-device implementation of `CellularPathMonitoring`: watches whether a cellular path is
available (airplane mode / no SIM / cellular off ⇒ unavailable) using
`NWPathMonitor(requiredInterfaceType: .cellular)`, driving the app's No-cellular state and
auto-resume.

## Do

Create `Sources/Live/CellularPathMonitor.swift`:

```swift
final class CellularPathMonitor: CellularPathMonitoring { init() }
```

- `availability()`: back the `AsyncStream<Bool>` with
  `NWPathMonitor(requiredInterfaceType: .cellular)`. Since iOS 17 `NWPathMonitor` IS an
  `AsyncSequence` of `NWPath` — consume it in a `Task` owned by the stream:
  `for await path in monitor { continuation.yield(path.status == .satisfied) }`.
  The monitor yields the current path promptly on subscription, satisfying the protocol's
  "current value first" contract. Cancel the task in `onTermination`; call `monitor.cancel()` too.
- Map ONLY `.satisfied` → `true`; `.unsatisfied`/`.requiresConnection` → `false`. Do not inspect
  `unsatisfiedReason` in v1.
- Wire the composition root: on DEVICE builds (no `-ScoutScenario` argument) use
  `CellularPathMonitor()`; simulator keeps `SimulatedPathMonitor` (there is no cellular interface in
  the simulator — the live monitor would permanently report false there).

## Done when

- The class compiles with the exact shape above and is wired in the device branch of
  `AppEnvironment` alongside the radio provider's branch from the previous task (same
  `#if !targetEnvironment(simulator)` + no-scenario-argument condition — extract a shared helper if
  that's cleaner).
- Simulator behavior unchanged: `SCOUT_LAUNCH_ARGS="-firstRunConsentGiven YES -ScoutScenario Great"
  ./build_run.sh` still shows the measuring screen (not paused).
- Standard DoD run green. Real airplane-mode behavior is human-verified at the device gate.
