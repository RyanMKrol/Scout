# T025 — 🔒 Human gate: on-device end-to-end verification (real cellular)

## Overview

The live measurement path (forced-cellular NetworkConnection, CoreTelephony, NWPathMonitor) can only
be truly verified on a physical iPhone with a SIM — CI and the simulator exercise none of it. This is
the single human gate that certifies the instrument actually works, including the P0 data-stop rule.

## Do

(Human, on a physical iPhone with an active SIM, app installed via Xcode run or TestFlight:)

1. **Forced cellular with Wi-Fi ON:** note Settings → Cellular → Scout's data usage; run Scout for
   ~2 minutes with Wi-Fi connected. The reading must move, and Scout's CELLULAR usage figure must
   grow by roughly the session counter's total — proof the transfer is pinned to cellular, not Wi-Fi.
2. **Data budget:** over that session, total (down + up) growth should land in the intended envelope
   — roughly 10–40 MB per minute. Note the actual figure in the worklog.
3. **Generation label:** matches the status-bar indicator (5G/LTE); if you can force LTE
   (Settings → Cellular → Voice & Data), the label follows within a few seconds.
4. **P0 — background stop:** note the usage counter, background the app (Home) for 60+ seconds,
   re-check Settings cellular usage: growth during background must be ~0. Repeat with screen lock.
5. **No-cellular fallback:** toggle Airplane Mode mid-session → PAUSED/"No cellular to measure"
   within a few seconds, dial goes idle; disable Airplane Mode → measuring resumes by itself.
6. **Consent flows:** delete + reinstall (or reset the flag) → splash → consent appears once;
   "Not now" → idle state, NO cellular usage accrues while idle; "Start sweeping" from idle →
   measuring; relaunch → consent not shown again.
7. **Instrument feel:** walk between a good-signal spot and a weak one (or a metal-shielded corner):
   the number/arc/color respond within ~2–3 s; bands feel sane (Great in a good spot, Poor where
   unusable). If the 6/2 thresholds or the 10/5 caps feel wrong in the real world, note it — that's
   a tuning follow-up, not a blocker.
8. **Comfort:** VoiceOver reads one combined element; largest Dynamic Type doesn't clip; Reduce
   Motion stops the sweep/pulse; phone shouldn't get hot in a 5-minute session.

## Done when

Every step above is performed and its observation recorded in `.harness/worklog/T025.md`; any
failures become new backlog tasks/ideas. Then `.harness/scripts/mark-done.sh T025`.
