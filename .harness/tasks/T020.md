# T020 — Accessibility: VoiceOver, Dynamic Type, Reduce Motion audit

## Overview

Make the instrument genuinely accessible per the handoff's accessibility section: the live dial reads
as ONE VoiceOver element with a polled combined value (never per-tick announcements), all screens
survive large Dynamic Type without clipping, and every decorative animation respects Reduce Motion.

## Do

1. **Combined VoiceOver element** — on the measuring screen's dial+footer region:
   - `.accessibilityElement(children: .ignore)`, `.accessibilityLabel("Current speed")`,
     `.accessibilityAddTraits(.updatesFrequently)` (VoiceOver POLLS an updatesFrequently element —
     this is the anti-spam mechanism; never fire per-tick announcements).
   - `.accessibilityValue(...)` built by a PURE function — create `Sources/Domain/AccessibilitySummary.swift`:

```swift
enum AccessibilitySummary {
    static func value(downloadMbps: Double, uploadMbps: Double, generation: RadioGeneration,
                      quality: SignalQuality, downloadBytes: Int64, uploadBytes: Int64) -> String
}
```

     Format: `"7 megabits per second down, 3 megabits per second up, 5G, Usable signal, 14 megabytes
     down and 6 megabytes up used this session"`. Rules: Mbps rounded to whole integers; readings at
     or above the caps phrase as "more than 10 megabits per second down" / "more than 5 … up";
     `.unknown` generation omits that clause; MB uses the same rounding as `megabytesDisplay` but
     spells "megabytes". Spell units out fully — VoiceOver mispronounces abbreviations.
   - Paused variants: a single element too — label "Measurement paused", value = the visible
     title + body copy.
2. **Buttons/labels**: consent + idle Start buttons keep their visible labels as accessibility
   labels; decorative views (radar mark, wedge, loader, pulsing dot, arrow glyphs) get
   `.accessibilityHidden(true)`.
3. **Dynamic Type**: cap the measuring screen at `.dynamicTypeSize(...DynamicTypeSize.accessibility2)`;
   verify (and fix any clipping) on splash, consent, measuring, and both paused variants at that cap —
   consent's content must scroll rather than overflow (per its spec), the dial scales via its
   ScaledMetrics, hero uses `.minimumScaleFactor(0.6)`/`.lineLimit(1)`.
4. **Reduce Motion audit**: confirm (and fix where missed) that Reduce Motion disables: splash/dial
   sweep rotation, the pulsing dot, and nothing else (number updates and arc-length changes are
   functional, they stay).

## Done when

- `Tests/AccessibilitySummaryTests.swift` covers: normal values (exact string above), capped values
  ("more than 10 megabits per second down"), unknown generation (clause omitted), zero bytes
  ("0.0 megabytes down and 0.0 megabytes up used this session" — matching megabytesDisplay rounding).
- Visual check: run `./build_run.sh` with `-firstRunConsentGiven YES -ScoutScenario Great`, then
  `xcrun simctl ui <sim> content_size accessibility-extra-large`, relaunch the app
  (`xcrun simctl launch` same args), screenshot with `xcrun simctl io <sim> screenshot`, and LOOK:
  nothing clipped or overlapping on the measuring screen. Reset with
  `xcrun simctl ui <sim> content_size large` afterwards (the build script assumes `large`).
- The accessibility modifiers from step 1–2 are present on the named views; standard DoD run green.
