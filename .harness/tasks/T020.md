# T020 — Accessibility application: VoiceOver elements, Dynamic Type, Reduce Motion audit

## Overview

Attach accessibility to the built screens per the handoff's accessibility section: the live dial
reads as ONE VoiceOver element whose value is the already-built `AccessibilitySummary` string
(polled, never per-tick announcements), all screens survive large Dynamic Type without clipping,
and every decorative animation respects Reduce Motion.

## Do

1. **Combined VoiceOver element** — on the measuring screen's dial+footer region:
   - `.accessibilityElement(children: .ignore)`, `.accessibilityLabel("Current speed")`,
     `.accessibilityAddTraits(.updatesFrequently)` (VoiceOver POLLS an updatesFrequently element —
     this is the anti-spam mechanism; never fire per-tick announcements).
   - `.accessibilityValue(AccessibilitySummary.value(...))` fed from the session.
   - Paused variants: a single element too — label "Measurement paused", value = the visible
     title + body copy.
2. **Buttons/labels**: consent + idle Start buttons keep their visible labels as accessibility
   labels; decorative views (radar mark, wedge, loader, pulsing dot, arrow glyphs) get
   `.accessibilityHidden(true)`.
3. **Dynamic Type**: cap the measuring screen at `.dynamicTypeSize(...DynamicTypeSize.accessibility2)`;
   verify (and fix any clipping) on splash, consent, measuring, and both paused variants at that
   cap — consent's content must scroll rather than overflow (per its spec); the dial and hero
   already carry ScaledMetrics.
4. **Reduce Motion audit**: confirm (and fix where missed) that Reduce Motion disables the
   splash/dial sweep rotation and the pulsing dot — and nothing else (number updates and arc-length
   changes are functional; they stay).

## Done when

- The modifiers from steps 1–2 are present on the named views (the summary string itself is already
  test-pinned by its own task).
- Visual check: run `./build_run.sh` with `-firstRunConsentGiven YES -ScoutScenario Great`, then
  `xcrun simctl ui <sim> content_size accessibility-extra-large`, relaunch the app
  (`xcrun simctl launch` same args), screenshot via `xcrun simctl io <sim> screenshot`, and LOOK:
  nothing clipped or overlapping on the measuring screen. Reset with
  `xcrun simctl ui <sim> content_size large` afterwards (the build script assumes `large`).
- Standard DoD run green.
