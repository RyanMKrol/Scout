# T029 — Foreground-only lifecycle (P0) + screenshot launch-arg plumbing

## Overview

Two small, related pieces of app-shell hardening: (1) the P0 rule — the moment Scout leaves the
foreground, ALL measurement stops immediately; (2) the launch-argument plumbing that lets the
harness's screenshot hook and UI tests force any screen/state.

## Do

1. **P0 lifecycle** — in `Sources/App/RootView.swift`:
   - Observe `@Environment(\.scenePhase)`. On `.inactive` and `.background`: call `session.stop()`
     IMMEDIATELY — comment it as the P0 rule (no data transfer outside the foreground, ever). On
     `.active`: call `session.start()` ONLY IF `router.phase == .measuring && router.consentGiven`.
   - This must compose with (not duplicate) the existing start-on-grant call from the previous task —
     one clear owner per transition.
2. **Force-idle state for tooling** — if `UserDefaults.standard.bool(forKey: "ScoutForceIdle")` is
   set at launch: skip the splash and land directly on measuring WITHOUT consent (router at
   `.measuring`, consent false → the stub's paused/idle branch). Implement in the RootView layer
   (e.g. call `router.splashFinished()` immediately and force the consent flag false for the run) —
   do NOT change AppRouter's frozen init.
3. **`build_run.sh` launch-args passthrough** (this file is scope-exempt): change the launch line to
   `xcrun simctl launch "$SIM" "$BUNDLE_ID" ${SCOUT_LAUNCH_ARGS:-}` (UNQUOTED expansion so multiple
   args split), enabling e.g.
   `SCOUT_LAUNCH_ARGS='-firstRunConsentGiven YES -ScoutScenario Great' ./build_run.sh`.

## Done when

- `SCOUT_LAUNCH_ARGS="-firstRunConsentGiven YES -ScoutScenario Great" ./build_run.sh` → screenshot
  shows the measuring stub with a nonzero reading; `-ScoutScenario None` → "PAUSED";
  `-ScoutForceIdle YES` → "PAUSED" + the Start button. LOOK at each screenshot.
- The scenePhase handler exists exactly as specced with the P0 comment; `bash -n build_run.sh`
  passes and a plain `./build_run.sh` (no args) still works.
- Standard DoD run green.
