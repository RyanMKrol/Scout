## Overview

Replace the measuring stub with the real home screen: status row ("SWEEPING" + pulsing dot), the
SweepDial fed live from the session, and the footer (quality word + split down/up session-data
counter). Per `design/design_handoff_scout/README.md` §3.

## Do

Create `Sources/Screens/MeasuringView.swift`, replacing `MeasuringStubView` in RootView (keep the
paused/idle branch logic working — its full-fidelity rendering is the NEXT task; for now route those
branches to the existing paused texts + `SweepDialView(mode: .idle)`).

Layout: full-screen `ScoutTheme.background`; VStack, space-between (status row top, dial center,
footer bottom); padding top ~120 pt, sides 30 pt, bottom 48 pt.

1. **Status row** (id `measuring.status`, centered): `PulsingDot(color: qualityColor)` (7 pt) + 9 pt
   gap + "SWEEPING" — 14 pt semibold, `.tracking(2.5)`, uppercase as written, `ScoutTheme.white(0.5)`.
2. **Dial**: `SweepDialView(mode: .live(...))` built from the injected `SweepSession`:
   - `downDisplay = ScoutMeter.downloadDisplay(session.downloadMbps)`, `upDisplay =
     ScoutMeter.uploadDisplay(session.uploadMbps)`;
   - `downFraction/upFraction` via the `ScoutMeter` arc-fraction functions;
   - `qualityColor`: `.great → ScoutTheme.great`, `.usable → ScoutTheme.usable`,
     `.poor → ScoutTheme.poor` (define a small `SignalQuality → Color` mapping helper in this file
     or DesignSystem);
   - `generationText = session.generation.rawValue`.
3. **Footer** (centered, 14 pt gap):
   - Quality word (id `measuring.quality`): "Great"/"Usable"/"Poor" — 20 pt semibold, qualityColor,
     color animated `.easeInOut(ScoutMotion.colorDuration)` keyed on the quality.
   - Counter row, 14 pt, `.monospacedDigit()`: down-arrow glyph +
     `ScoutMeter.megabytesDisplay(bytes: session.sessionDownloadBytes)` + "down" (value+unit in
     `ScoutTheme.white(0.72)`, id `measuring.dataDown`) · centered dot separator
     (`ScoutTheme.white(0.3)`) · up-arrow glyph + `megabytesDisplay` of upload bytes + "up" (in
     `ScoutTheme.uploadText`, id `measuring.dataUp`).
   - Caption beneath: "used this session" — 12 pt, `ScoutTheme.white(0.32)`.
4. **4 Hz hygiene**: the session updates several times per second — keep `PulsingDot` and the dial's
   wedge in stable leaf views (already built that way); this screen only recomputes cheap text/values.
   No `GeometryReader`; fixed design metrics.
