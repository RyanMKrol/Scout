# T050 — Session data/time auto-stop backstop

## Overview
A continuous unbounded download (T045) uses a lot of data — potentially hundreds of MB per minute on a
fast connection. To prevent a forgotten sweep from silently burning gigabytes, add a backstop that
auto-stops the sweep after a session data or time cap, with a clear dismissible warning that lets the
user continue if they meant to. Pause/stop remain the primary controls; this is a safety net.

## Do
In `SweepSession` (`Sources/Domain/SweepSession.swift`), track elapsed session time (via the injected
clock) and total session bytes (`sessionDownloadBytes` + `sessionUploadBytes`) while measuring, and when
either crosses a configurable cap (named constants — default e.g. ~500 MB or ~5 minutes) automatically
`stop()` the sweep and set an observable "auto-stopped, backstop reached" state. In the measuring UI
(`Sources/Screens/**`, `Sources/App/**`), when that state is set, present a clear dismissible warning
that explains the sweep was auto-stopped to protect data use and offers to resume/continue (dismiss →
restart the sweep and reset the backstop counters for a fresh allowance). Give the warning and its
continue control stable `accessibilityIdentifier`s. The backstop must not fire in the paused state
(paused already isn't consuming data) and must not affect the default no-cap-reached flow.

## Done when
- While measuring, crossing the session data cap OR the time cap auto-stops the sweep and surfaces an
  observable backstop state; neither cap firing is possible while paused or stopped.
- The measuring UI shows a dismissible warning (stable `accessibilityIdentifier`) explaining the
  auto-stop and offering to continue; continuing resets the backstop allowance and resumes measuring.
- `Tests/SweepSessionTests.swift` proves, with the injected clock + fake: advancing bytes past the data
  cap (and, separately, time past the time cap) triggers the auto-stop state and halts sampling; a
  continue resets the counters; the caps never fire while paused.
- On device (manual, T025-style): leaving a sweep running eventually auto-stops with the warning rather
  than downloading indefinitely.
