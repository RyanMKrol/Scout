# T033 — AccessibilitySummary: the spoken-value builder (pure)

## Overview

The pure function that composes the VoiceOver value string for the live measuring screen — one
combined, fully-spelled-out sentence covering both directions, generation, quality, and session
data. Built and tested standalone; the NEXT accessibility task attaches it to the UI.

## Do

Create `Sources/Domain/AccessibilitySummary.swift`:

```swift
enum AccessibilitySummary {
    static func value(downloadMbps: Double, uploadMbps: Double, generation: RadioGeneration,
                      quality: SignalQuality, downloadBytes: Int64, uploadBytes: Int64) -> String
}
```

Format: `"7 megabits per second down, 3 megabits per second up, 5G, Usable signal, 14 megabytes
down and 6 megabytes up used this session"`. Rules:

- Mbps rounded to whole integers for speech (even though the display shows a decimal).
- Readings at or above the caps phrase as "more than 10 megabits per second down" /
  "more than 5 megabits per second up" (`ScoutMeter.downloadCapMbps` / `.uploadCapMbps`).
- `.unknown` generation omits that clause entirely (no dangling comma).
- Quality names: "Great signal" / "Usable signal" / "Poor signal".
- Megabyte figures reuse `ScoutMeter.megabytesDisplay` rounding but spell the unit out
  ("14 megabytes", "3.2 megabytes") — VoiceOver mispronounces abbreviations, so no "MB", no "Mbps".

## Done when

- `Tests/AccessibilitySummaryTests.swift` covers: normal values (the exact example string above,

  ("0.0 megabytes down and 0.0 megabytes up used this session").
