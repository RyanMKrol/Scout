# T018 — HTTP probe support: request building, incremental response parsing, probe pacing (pure)

## Overview

The pure plumbing the live cellular sampler needs: build raw HTTP/1.1 probe requests (Scout speaks
HTTP/1.1 by hand over a TLS `NetworkConnection` — the iOS 26 Network stack has NO HTTP application
protocol), incrementally parse response bytes arriving in arbitrary chunks on a persistent keep-alive
connection, and compute probe pacing. Fully unit-tested; zero networking in this task.

## Do

Create `Sources/Live/HTTPProbe.swift` containing:

1. **Request building**:

```swift
enum HTTPProbeRequest {
    static let host = "speed.cloudflare.com"
    static let probeBytes = 262_144            // 256 KiB per probe, both directions

    static func download() -> Data
    static func uploadHeader() -> Data          // headers only; body sent separately
    static func uploadBody() -> Data            // probeBytes of 0x55
}
```

   - `download()`: `"GET /__down?bytes=262144 HTTP/1.1\r\nHost: speed.cloudflare.com\r\nConnection: keep-alive\r\n\r\n"` as UTF-8.
   - `uploadHeader()`: `"POST /__up HTTP/1.1\r\nHost: speed.cloudflare.com\r\nContent-Type: application/octet-stream\r\nContent-Length: 262144\r\nConnection: keep-alive\r\n\r\n"`.

2. **Incremental response parser** — consumes bytes as they arrive (any chunking), across MULTIPLE
   sequential responses on one connection:

```swift
struct HTTPResponseParser {
    enum Event: Equatable {
        case needMoreData
        case headersComplete(statusCode: Int, contentLength: Int?)
        case bodyProgress(newBodyBytes: Int)     // body bytes in the chunk just fed
        case responseComplete                    // current response fully consumed; parser resets
    }
    enum ParserError: Error, Equatable { case malformedResponse, nonSuccessStatus(Int) }

    mutating func feed(_ data: Data) throws -> [Event]
}
```

   - Parse the status line (`HTTP/1.1 200 OK` → 200). A status other than 2xx →
     `throw .nonSuccessStatus(code)` (the sampler treats it as a failed probe — Cloudflare 403s
     oversized requests, and measuring an error body as throughput would corrupt readings).
   - Headers end at `\r\n\r\n`; extract `Content-Length` case-insensitively. A missing
     Content-Length → treat as malformed for Scout's purposes (both endpoints send it).
   - Body: count bytes until Content-Length consumed → emit `responseComplete` and RESET so the next
     bytes belong to the next response (keep-alive pipelining of sequential probes).
   - One `feed` call may produce several events (e.g. headers + body + complete in a single chunk),
     and leftover bytes after `responseComplete` must roll into the next response's parsing.

3. **Pacer** — pure arithmetic:

```swift
enum ProbePacer {
    static let interval: Duration = .milliseconds(500)
    static func delayBeforeNextProbe(lastProbeStartedAt: ContinuousClock.Instant?,
                                     now: ContinuousClock.Instant) -> Duration   // .zero if overdue / first
}
```

## Done when

- Tests (e.g. `Tests/HTTPProbeTests.swift`) cover at least:
  - request builders produce byte-exact strings above; `uploadBody().count == 262_144`;
  - a full 200 response fed in ONE chunk → `[headersComplete(200, N), bodyProgress(N),
    responseComplete]`;
  - the same response fed byte-by-byte (loop) accumulates the same totals and ends with
    `responseComplete`;
  - a chunk boundary splitting the `\r\n\r\n` and one splitting the status line both parse;
  - two back-to-back keep-alive responses in one buffer → two `responseComplete`s with correct
    per-response body counts;
  - `403` status → throws `.nonSuccessStatus(403)`; garbage first line → `.malformedResponse`;
  - pacer: nil last-start → `.zero`; 200 ms since last → 300 ms; ≥ 500 ms since last → `.zero`.
- No sockets/URLSession/Network imports in this file (pure Foundation); standard DoD run green.
