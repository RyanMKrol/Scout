# T013 — SweepDial: the dual-arc radar dial assembly

## Overview

Assemble the 288 pt radar dial from the existing building blocks: rotating sweep wedge
(SweepWedgeView), outer download track + arc (quality-colored), inner upload track + arc (fixed cool
blue), and the DialCenterStack in the middle. Per `design/design_handoff_scout/README.md` §3 (dial
spec) and §4 (idle rendering). Wired into the measuring stub in place of the bare center stack.

## Do

1. Create `Sources/DesignSystem/SweepDialView.swift` with EXACTLY this input surface (binding to
   the existing `DialContent` and reusing `DialCenterStack` internally):

```swift
struct SweepDialView: View {
    enum Mode: Equatable { case live(DialContent), idle }
    init(mode: Mode)
}
```

2. Layered construction, 288×288 pt frame (ZStack):
   - **Sweep wedge** (live only, hidden under Reduce Motion): `SweepWedgeView(color: qualityColor,
     opacity: 0.14, wedgeDegrees: 92, period: ScoutMotion.dialSweepPeriod, rotating: true)`, 258 pt.
   - **Download track**: circle diameter 264 (r 132), stroke 7 pt, `ScoutTheme.white(0.07)`
     (idle: 0.06).
   - **Download arc** (live only): same geometry, `Circle().trim(from: 0, to: downFraction)`,
     `StrokeStyle(lineWidth: 7, lineCap: .round)`, `.rotationEffect(.degrees(-90))` (fills from the
     top), `.foregroundStyle(qualityColor)`. TWO coexisting keyed animations:
     `.animation(.linear(duration: ScoutMotion.arcDuration), value: downFraction)` and
     `.animation(.easeInOut(duration: ScoutMotion.colorDuration), value: qualityColor)` — each
     fires only when its own value changes.
   - **Upload track** (live only): circle diameter 200 (r 100), stroke 6 pt, `ScoutTheme.white(0.06)`.
   - **Upload arc** (live only): trim to `upFraction`, 6 pt round cap, −90°, fixed
     `ScoutTheme.uploadArc` (NO color animation — upload color never signals quality), arc-length
     animation `.linear(ScoutMotion.arcDuration)` keyed on `upFraction`.
   - **Center**: `DialCenterStack(mode:)` — `.live(content)` / `.idle` passthrough.
3. **Dynamic Type**: `@ScaledMetric(relativeTo: .largeTitle)` on the dial diameter (base 288) so the
   instrument scales with the hero text; the component must not clip at `.accessibility2`.
4. **4 Hz hygiene**: the wedge keeps stable identity (never inside an `if` that flips per tick);
   per-tick changes flow only through `DialContent` values.
5. Wire into `MeasuringStubView`: replace the bare `DialCenterStack` with
   `SweepDialView(mode: .live(content))` (same content construction), and the PAUSED branch's idle
   center with `SweepDialView(mode: .idle)`.
