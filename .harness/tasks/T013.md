# T013 — SweepDial: the dual-arc radar dial component

## Overview

The centerpiece component: a 288 pt radar dial with an outer download arc (quality-colored), an inner
upload arc (fixed cool blue), a rotating sweep wedge, and the center stack (generation, 80 pt hero
download number, "↓ Mbps down" row, "↑ n.n up" row). Per `design/design_handoff_scout/README.md` §3
(dial spec) and §4 (idle rendering). Wired into the measuring stub so it's visible on screen.

## Do

1. Create `Sources/DesignSystem/SweepDialView.swift` with EXACTLY this input surface:

```swift
struct SweepDialView: View {
    struct LiveContent: Equatable {
        let downDisplay: String        // e.g. "7.4" / "10+"
        let upDisplay: String          // e.g. "3.1" / "5+"
        let downFraction: Double       // ScoutMeter.downloadArcFraction output
        let upFraction: Double         // ScoutMeter.uploadArcFraction output
        let qualityColor: Color
        let generationText: String     // "" hides the generation label
    }
    enum Mode: Equatable { case live(LiveContent), idle }
    init(mode: Mode)
}
```

2. Layered construction, 288×288 pt fixed frame (ZStack):
   - **Sweep wedge** (live only, hidden under Reduce Motion): `SweepWedgeView(color: qualityColor,
     opacity: 0.14, wedgeDegrees: 92, period: ScoutMotion.dialSweepPeriod, rotating: true)`, 258 pt.
   - **Download track**: circle diameter 264 (r 132), stroke 7 pt, `ScoutTheme.white(0.07)`
     (idle: 0.06).
   - **Download arc** (live only): same geometry, `Circle().trim(from: 0, to: downFraction)`,
     `StrokeStyle(lineWidth: 7, lineCap: .round)`, `.rotationEffect(.degrees(-90))` (fills from top),
     `.foregroundStyle(qualityColor)`. TWO coexisting keyed animations: `.animation(.linear(duration:
     ScoutMotion.arcDuration), value: downFraction)` and `.animation(.easeInOut(duration:
     ScoutMotion.colorDuration), value: qualityColor)` — each fires only when its value changes.
   - **Upload track**: circle diameter 200 (r 100), stroke 6 pt, `ScoutTheme.white(0.06)` (live only).
   - **Upload arc** (live only): trim to `upFraction`, 6 pt round cap, −90°, fixed
     `ScoutTheme.uploadArc` (NO color animation — upload color never signals quality), arc-length
     animation `.linear(ScoutMotion.arcDuration)` keyed on `upFraction`.
   - **Center stack** (live): generation label (15 pt, weight .medium, `.tracking(2)`,
     `ScoutTheme.white(0.5)`, id `measuring.generation`, hidden when empty) → hero download number
     (80 pt, weight .semibold, `.tracking(-3)`, `.monospacedDigit()`, `qualityColor` with the same
     0.45 s ease color animation, id `measuring.hero`) → row [down-arrow glyph + "Mbps down"] (16 pt
     medium, `ScoutTheme.white(0.55)`) → row [up-arrow glyph + upDisplay (19 pt semibold,
     `ScoutTheme.uploadText`, `.monospacedDigit()`, id `measuring.upload`) + "up" (14 pt,
     `ScoutTheme.uploadText.opacity(0.7)`)].
   - **Center stack (idle)**: "—" em dash (96 pt, weight .semibold, `ScoutTheme.white(0.22)`) over
     "Mbps" (18 pt medium, `ScoutTheme.white(0.35)`). No arcs, no wedge, download track only at 0.06.
   - **Arrow glyphs**: a small `ArrowGlyph(direction:)` shape in the same file — stroked `Path`
     ~11×13 pt, 1.5 pt line width, round caps/joins: vertical shaft plus two 45° head strokes
     (down: head at bottom; up: same path flipped). Match the prototype's glyph (`M6.5 1v9
     M6.5 10.5 L3 7 M6.5 10.5 L10 7` in a 13×14 viewBox, scaled).
3. **Dynamic Type**: `@ScaledMetric(relativeTo: .largeTitle)` for the hero size (88→80 base) AND the
   dial diameter so the instrument scales together; `.minimumScaleFactor(0.6)` + `.lineLimit(1)` on
   the hero; the measuring screen will cap overall size — the component itself must not clip at
   `.accessibility2`.
4. **Isolation for 4 Hz updates**: the wedge and any repeat-forever animation live in leaf subviews
   with stable identity (never inside an `if` that flips per tick); per-tick changes flow only
   through `LiveContent` values.
5. Wire into `MeasuringStubView`: when measuring normally, show
   `SweepDialView(mode: .live(...))` built from the session via `ScoutMeter` display/fraction
   functions and `ScoutTheme` quality color mapping (`great→.great, usable→.usable, poor→.poor`);
   keep the stub's PAUSED branch, now rendering `SweepDialView(mode: .idle)` beneath the existing
   paused texts.
