# T036 — 🔒 TestFlight / App Store submission

## Overview

Ship v1: get Scout onto TestFlight (and, when you choose, into App Store review). Everything here
is human-only — Apple account actions, review answers, and store metadata judgment calls.

## Do

(Human, interactively — prerequisites: T023 icon approved, T025 device E2E passed, T026 design
fidelity accepted:)

1. **Membership:** paid Apple Developer Program membership active.
2. **App record:** App Store Connect → create the app (bundle id `com.ryankrol.scout`, name "Scout"
   — have fallbacks ready, the name may be taken; SKU; primary language).
3. **Archive + upload:** from the repo: `xcodegen generate`, then Xcode → Product → Archive (Any iOS
   Device) → Distribute → App Store Connect. (Signing from T035; bump `CFBundleVersion` in
   `project.yml` for each upload.)
4. **TestFlight:** add yourself (internal tester), install via the TestFlight app, sanity-check the
   real-cellular experience end to end once more.
5. **Store metadata:** description (lead with the honest framing: measures real usable speed, not
   signal bars — mirrors the README), keywords, support URL, screenshots (capture from the
   simulator/device per Apple's size requirements), category (Utilities), age rating
   questionnaire, privacy "Data Not Collected" declaration (matches `PrivacyInfo.xcprivacy`),
   export compliance already answered by `ITSAppUsesNonExemptEncryption: false`.
6. **Review notes:** explain that the app measures cellular throughput by downloading/uploading
   over the cellular interface (reviewers on Wi-Fi-only devices will see the No-cellular state —
   tell them that's expected behavior, not a bug).
7. Submit for review when satisfied; record the outcome (and any rejection feedback as new ideas).

## Done when

A TestFlight build is installed and working on your device; App Store submission is either sent or
deliberately deferred with the reason noted in the worklog. Then
`.harness/scripts/mark-done.sh T036`.
