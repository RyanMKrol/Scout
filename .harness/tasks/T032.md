# T032 — HTTP probe requests + probe pacing (pure)

## Overview

The small pure pieces of the live measurement engine: byte-exact HTTP/1.1 probe request builders for
Cloudflare's speed endpoints (Scout speaks HTTP/1.1 by hand over a TLS `NetworkConnection` — the
iOS 26 Network stack has NO HTTP application protocol), and the pacing arithmetic that spaces probes.
No networking, fully unit-tested. (The response PARSER is a separate task, T018.)

## Do

Create `Sources/Live/HTTPProbeRequest.swift`:

```swift
enum HTTPProbeRequest {
    static let host = "speed.cloudflare.com"
    static let probeBytes = 262_144            // 256 KiB per probe, both directions

    static func download() -> Data
    static func uploadHeader() -> Data          // headers only; body sent separately
    static func uploadBody() -> Data            // probeBytes of 0x55
}
```

- `download()`: `"GET /__down?bytes=262144 HTTP/1.1\r\nHost: speed.cloudflare.com\r\nConnection: keep-alive\r\n\r\n"` as UTF-8.
- `uploadHeader()`: `"POST /__up HTTP/1.1\r\nHost: speed.cloudflare.com\r\nContent-Type: application/octet-stream\r\nContent-Length: 262144\r\nConnection: keep-alive\r\n\r\n"`.

And `Sources/Live/ProbePacer.swift` — pure arithmetic:

```swift
enum ProbePacer {
    static let interval: Duration = .milliseconds(500)
    static func delayBeforeNextProbe(lastProbeStartedAt: ContinuousClock.Instant?,
                                     now: ContinuousClock.Instant) -> Duration   // .zero if overdue / first
}
```

## Done when

- Tests (e.g. `Tests/HTTPProbeRequestTests.swift`) cover:
  - both request builders produce the byte-exact strings above (compare against literal strings);
  - `uploadBody().count == 262_144` and every byte is `0x55`;
  - pacer: nil last-start → `.zero`; 200 ms since last → 300 ms remaining; ≥ 500 ms since last →
    `.zero` (fabricate instants from one `ContinuousClock().now` base via `advanced(by:)`).
- No sockets/URLSession/Network imports (pure Foundation); standard DoD run green.
