# T034 — Dark-mode-only enforcement + dark launch screen

## Overview

The design is dark-mode only by decision, but nothing enforces it: with the device in light mode,
system surfaces render light and the empty `UILaunchScreen` flashes WHITE on cold launch before the
near-black splash appears. Pin the app to dark and give the launch screen the design background.

## Do

1. **`project.yml`** — in the Scout target's `info.properties`:
   - add `UIUserInterfaceStyle: Dark` (locks the app to dark appearance regardless of the system
     setting — status bar, alerts, system sheets all follow);
   - change `UILaunchScreen: {}` to `UILaunchScreen: { UIColorName: LaunchBackground }` (keep the
     key present — an ABSENT UILaunchScreen re-triggers legacy letterboxing).
2. **Launch color asset** — add `Sources/Assets.xcassets/LaunchBackground.colorset/Contents.json`:
   a universal sRGB color equal to the design background `#050506` (components red `0x05`, green
   `0x05`, blue `0x06`, alpha 1.0), standard Xcode colorset JSON shape.
3. **Belt-and-braces in SwiftUI** — add `.preferredColorScheme(.dark)` on RootView's top-level
   content.

## Done when

- After `xcodegen generate` and a build, the built app's Info.plist confirms both keys:
  `plutil -p build/Debug-iphonesimulator/Scout.app/Info.plist` shows `UIUserInterfaceStyle => Dark`
  and `UILaunchScreen => { UIColorName => LaunchBackground }` (run `./build_run.sh` first to
  produce the build; record the plutil output in the worklog).
- `./build_run.sh` screenshot still renders the app full-screen and dark (no letterbox regression —
  the UILaunchScreen key must remain effective).
- The colorset is committed and the standard DoD run is green.
