# T022 — App icon: headless generation script + asset catalog

## Overview

Produce the app icon from the design spec (the radar mark on a near-black superellipse — see
`design/design_handoff_scout/README.md` "Assets" + the prototype's 3b icon panel) with a committed,
re-runnable CoreGraphics script — no design tools, no Xcode GUI. A human reviews the result in the
next (gated) task.

## Do

1. **`icontool/generate_icon.swift`** — a standalone script run as
   `swift icontool/generate_icon.swift` on macOS (CoreGraphics + ImageIO only), writing a
   1024×1024 **opaque** PNG (no alpha — App Store requirement) to
   `Sources/Assets.xcassets/AppIcon.appiconset/AppIcon.png`. Draw, scaled from the design's 176 pt
   reference frame (scale factor 1024/176 ≈ 5.818):
   - Background: SQUARE (no rounded corners — iOS applies the superellipse mask) radial gradient,
     center at (30%, 25%) from top-left, `#101015` at center → `#050506` at 70%+ radius.
   - Sweep wedge: angular (conic) gradient centered mid-icon: transparent until 250°, ramping to
     `#63D99B` at 55% alpha by 358°, back to transparent at 360° (composite OVER the opaque
     background — the FILE stays opaque).
   - Rings (stroked circles, centered): r 66 → 384 px, stroke 3 → 17.5 px, white at 14% alpha;
     r 42 → 244 px, same stroke, white 18%.
   - Center dot: filled circle r 10 → 58 px, `#5FE19E` (the icon's slightly brighter green —
     oklch(0.82 0.15 158), pre-converted).
   - Deterministic: same input → byte-identical output where CG allows; no randomness, no timestamps.
2. **Asset catalog** — create `Sources/Assets.xcassets/` with:
   - `Contents.json`: `{ "info": { "author": "xcode", "version": 1 } }`
   - `AppIcon.appiconset/Contents.json`: the single-size universal form —
     one image entry `{ "filename": "AppIcon.png", "idiom": "universal", "platform": "ios",
     "size": "1024x1024" }` plus the standard info block. (Dark/tinted variants are optional on
     iOS — deliberately skipped; the near-black icon self-darks.)
   - The generated `AppIcon.png`, committed.
3. **`project.yml`**: add `ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon` to the Scout target's
   settings.
4. Run the script, run `xcodegen generate`, and confirm the built app carries the icon (build via
   the standard DoD; additionally `./build_run.sh` then
   `xcrun simctl io <sim> screenshot` is NOT sufficient to see the home-screen icon — instead verify
   the asset compiled by checking the built .app contains `Assets.car`, and OPEN the generated
   `AppIcon.png` file itself and LOOK at it: rings visible, green dot centered, soft green wedge in
   the upper area, near-black ground, no transparency artifacts).

## Done when

- `swift icontool/generate_icon.swift` regenerates the committed PNG; the PNG is 1024×1024, opaque
  (no alpha channel), and visually matches the description (you looked at it).
- The asset catalog compiles into the app (DoD build green with the catalog present) and
  `ASSETCATALOG_COMPILER_APPICON_NAME` is set.
- Script, Contents.json files, and PNG are all committed together.
