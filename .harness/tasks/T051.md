# T051 — Honest data-use copy for the continuous-download model

## Overview
The continuous-download redesign (T045) makes data use high and variable (it scales with your
connection speed), superseding the "~30–60 MB/min" figure T038 wrote and the earlier "10–40 MB/min"
claim. The first-run consent copy and the README must tell the truth so the app doesn't promise a data
budget it no longer honors.

## Do
Update the first-run consent copy in `Sources/Screens/ConsentView.swift` (the data-use line, ~line 40,
currently a bold MB-per-minute figure) and every data-use reference in `README.md` to describe the real
behaviour: Scout streams continuously while sweeping and uses **as much cellular data as your
connection allows — potentially hundreds of MB per minute on a fast connection**, that it stops when you
pause or stop, and that it auto-stops after a safety cap (the T050 backstop). Do not keep any specific
low "10–40" / "30–60 MB per minute" figure that the continuous model contradicts. Keep the tone honest
and non-alarming, consistent with the existing consent voice.

## Done when
- The consent screen's data-use line and all README data-use references describe continuous, high,
  connection-dependent data use (no stale fixed "10–40" / "30–60 MB/min" figure remains —
  `grep -rn "MB per minute\|MB/min" Sources README.md` shows only the updated wording).
- The copy mentions that pausing/stopping halts data use and that a safety auto-stop exists.
- The consent screen builds and renders with the new copy on Scout-Sim.

## Notes
- Do NOT edit `.harness/**`. Retiring the outdated data-budget rows in the limitation ledger is the
  human integrator's job at merge.
