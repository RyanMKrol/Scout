# T018 ‚Äî Incremental HTTP/1.1 response parser (pure)

## Outcome

**Status:** done

## What was done

Implemented `HTTPResponseParser.swift` ‚Äî a pure, incremental HTTP/1.1 response parser that:

- Parses status lines and extracts status codes (2xx only; non-2xx throw `nonSuccessStatus`)
- Extracts headers case-insensitively (specifically `Content-Length`)
- Tracks body consumption against `Content-Length` and resets for keep-alive pipelining
- Emits events: `needMoreData`, `headersComplete`, `bodyProgress`, `responseComplete`
- Handles arbitrary chunk boundaries (partial headers, partial body, complete responses in one chunk)
- No networking imports; pure Foundation only

## Tests written

`HTTPResponseParserTests.swift` covers 18 test cases:

- Full response in one chunk
- Byte-by-byte feeding with body byte accumulation
- Chunk boundaries splitting status line and header delimiter
- Two back-to-back keep-alive responses
- Error cases (403, malformed status line)
- Header case-insensitivity
- Zero-length responses
- Partial body consumption across multiple chunks
- Multiple headers with Content-Length extraction
- Parser reset after each response
- Status code edge cases (201, 299, 300, 100)
