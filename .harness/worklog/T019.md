# T019 — Live cellular throughput sampler (NetworkConnection, forced .cellular)

## 2026-07-17

Did:
- Added `Sources/Live/CellularThroughputSampler.swift`: a `ThroughputSampling` conformance that
  builds a `NetworkConnection<TLS>` pinned to `speed.cloudflare.com:443` via
  `.parameters { TLS() }.requiredInterfaceType(.cellular).multipathServiceType(.disabled)` — the
  exact construction from the task spec (verified against the installed iOS 26.5 SDK's
  `Network.swiftinterface`: `NetworkConnection(to:using:)`, `receive(atLeast:atMost:) ->
  ApplicationProtocol.Message<Data>` where `Message<T> = (content: T, metadata: Self.Metadata)`,
  and the `StreamProtocol`/`OneToOneProtocol` `send`/`receive` async APIs all matched).
- Probe loop runs inside the `AsyncStream`'s owned `Task` (cancelled from `onTermination`),
  alternating download/upload probes paced by `ProbePacer.delayBeforeNextProbe`, built from the
  existing pure pieces (`HTTPProbeRequest`, `HTTPResponseParser`) from T018/T032. On any
  send/receive/parse error the connection is dropped, the loop sleeps 1s (cancellable), builds a
  fresh `NetworkConnection`, and continues — no sample is yielded for a failed probe and the
  stream never crashes.
- Wired `AppEnvironment`'s device (non-simulator) branch: `.live` scenario now returns
  `CellularThroughputSampler()`; simulator and other forced scenarios keep `SimulatedSampler`
  unchanged.
