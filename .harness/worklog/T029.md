# T029 — Foreground-only lifecycle (P0) + screenshot launch-arg plumbing

## 2026-07-17

**Built and verified cold.**

### What was done

1. **P0 lifecycle in RootView.swift:**
   - Added `@Environment(\.scenePhase)` observer
   - On `.inactive` and `.background`: call `session.stop()` immediately with P0 rule comment
   - On `.active`: call `session.start()` only if `router.phase == .measuring && router.consentGiven`
   - Composes cleanly with the existing start-on-grant call in `.task`

2. **Force-idle state for tooling:**
   - Added `@State private var forceIdle: Bool` and read from `UserDefaults.standard.bool(forKey: "ScoutForceIdle")`
   - In init: set `consentGiven: initialConsent || forceIdle` so `splashFinished()` lands on measuring
   - In body: pass `router.consentGiven && !forceIdle` to MeasuringStubView, so forceIdle shows paused branch
   - In task: on forceIdle, call `splashFinished()` and set `consentGiven = false`

3. **`build_run.sh` launch-args passthrough:**
   - Changed launch line from `xcrun simctl launch "$SIM" "$BUNDLE_ID"` to `xcrun simctl launch "$SIM" "$BUNDLE_ID" ${SCOUT_LAUNCH_ARGS:-}`
   - Unquoted expansion allows multiple args to split correctly
