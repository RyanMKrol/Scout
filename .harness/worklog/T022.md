# T022 — App icon: headless generation script + asset catalog

## Outcome

**Status:** done

## What was done

- `icontool/generate_icon.swift` — a standalone CoreGraphics + ImageIO script
  (`swift icontool/generate_icon.swift`) that renders `AppIcon.png` deterministically:
  - Opaque 1024×1024 square background: radial gradient, center (30%, 25%) from top-left,
    `#101015` → `#050506` by 70% radius, computed per-pixel.
  - Sweep wedge: angular gradient centered mid-icon (angle measured clockwise from top, matching
    `SweepWedgeView`'s `AngularGradient(center: .center)` convention) — transparent until 250°,
    ramping to `#63D99B` at 55% alpha by 358°, back to transparent at 360°, alpha-composited OVER
    the opaque background per-pixel.
  - Two stroked rings (r 384/244 px, stroke 17.5 px, white 14%/18% alpha) and a filled center dot
    (r 58 px, `#5FE19E`) drawn as vector CG paths on top, in a flipped coordinate space matching
    the per-pixel top-left screen convention.
  - Final image rendered through a `noneSkipLast` (no-alpha) CGContext so the written PNG has no
    alpha channel, per the App Store's opaque-icon requirement.
- `Sources/Assets.xcassets/Contents.json` + `AppIcon.appiconset/Contents.json` (single-size
  universal 1024×1024 entry) + the generated, committed `AppIcon.png`.
- `project.yml`: added `ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon` to the Scout target.
