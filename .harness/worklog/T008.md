# T008 — AppRouter: app-phase state machine (pure)

## Completion — 2026-07-17

### What was done

Created `Sources/App/AppRouter.swift` implementing a pure, observable state machine for app lifecycle phases:
- **Phase enum:** splash, consent, measuring
- **State transitions:** 
  - `splashFinished()` → navigates from splash to consent (if no consent) or measuring (if consent given), or stays at splash if `holdSplash=true`
  - `startSweeping()` → sets `consentGiven=true` and moves to measuring phase (valid from consent or measuring)
  - `declineConsent()` → moves to measuring with consent still false (only valid from consent)
- **Observable:** marked `@MainActor @Observable` for SwiftUI integration
- **Pure:** no persistence, no timers, no UserDefaults access

### Tests added: `Tests/AppRouterTests.swift`

Comprehensive test coverage of all transitions (10 tests):
- Fresh install flow: splash → consent → measuring with consent granted
- Returning user flow: splash → measuring (consent already given)
- Declined consent flow: consent → measuring (idle state) → can grant consent later via `startSweeping()`
- `holdSplash` behavior: `splashFinished()` is a no-op when flag is true
- Out-of-phase calls: `splashFinished()` twice, `declineConsent()` from measuring — all no-ops
- Observability: `consentGiven` property changes are tracked
- Edge case: `startSweeping()` always grants consent and moves to measuring
