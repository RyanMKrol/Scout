# T027 worklog

## 2026-07-17 — Simulated providers (scripted async streams)

### What was done

Added `Sources/Domain/SimulatedProviders.swift` with the three T006-seam simulated
implementations used in the Simulator, UI tests, and screenshot checks:

- `SimulatedSampler: ThroughputSampling` — a `Task` owned by the returned `AsyncStream`
  sleeps 250 ms per tick and alternates emitting a download sample (driven by
  `ScenarioWalk.nextDownloadMbps`) then an upload sample (`ScenarioWalk.uploadMbps(forDownload:)`),
  starting with download. Each sample uses `transferDuration = .milliseconds(180)`,
  `byteCount = Int(mbps × 1_000_000 × 0.18 / 8)`, and `endedAt = ContinuousClock().now` at
  emission time, using `SystemRandomNumberGenerator`. `.none` yields nothing and the stream
  stays open. `continuation.onTermination` cancels the backing task.
- `SimulatedRadioProvider: RadioInfoProviding` — yields one `RadioGeneration` immediately per
  scenario (`.great`/`.usable`/`.live` → `.fiveG`, `.poor` → `.lte`, `.none` → `.unknown`) then
  keeps the stream open.
- `SimulatedPathMonitor: CellularPathMonitoring` — yields `false` for `.none`, `true` otherwise,
  then keeps the stream open.

All three are declared without explicit actor isolation so they inherit the `Scout` target's
`SWIFT_DEFAULT_ACTOR_ISOLATION: MainActor` build setting (matching the existing `SweepSession`
seam conformers), which is why the new test file marks its `XCTestCase` `@MainActor`.

### Tests

Added `Tests/SimulatedProvidersTests.swift` (5 tests, all with bounded waits):
- `testGreatScenarioEmitsAlternatingSamples` — collects the first 4 samples from
  `SimulatedSampler(scenario: .great)` and asserts alternating `.download`/`.upload` directions
  starting with `.download`, positive byte counts, and 180 ms durations.
- `testNoneScenarioYieldsNoSamples` — confirms `.none` produces zero samples within a 400 ms
  window.
- `testRadioProviderYieldsExpectedGenerationPerScenario` — asserts the first yielded generation
  for every scenario.
- `testPathMonitorYieldsAvailabilityPerScenario` — asserts `.none` → `false`, `.great` → `true`.
- `testCancellingConsumerTaskStopsFurtherEmission` — cancels the consuming `Task` after the first
  sample and confirms no further samples arrive within a 600 ms grace window (pins the
  `onTermination` cancellation path).
