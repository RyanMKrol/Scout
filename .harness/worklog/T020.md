# T020 — Accessibility application: VoiceOver, Dynamic Type, Reduce Motion audit

**Built:** 2026-07-17

## Summary

Implemented comprehensive accessibility features across Scout's screens:

1. **Combined VoiceOver element** on the measuring screen's dial+footer region:
   - Wrapped footer in a single accessibility element (children: .ignore)
   - Set label to "Current speed" with `.updatesFrequently` trait (VoiceOver polls, no per-tick announcements)
   - Bound value to `AccessibilitySummary.value(...)` fed from the session
   - Added equivalent for paused variants: label "Measurement paused" with visible title + body as value

2. **Hidden decorative views** from accessibility:
   - ArrowGlyph instances (down/up arrows in footer and dial center) — `.accessibilityHidden(true)`
   - Separator dot in footer — `.accessibilityHidden(true)`
   - Status dot (pulsing or static) — `.accessibilityHidden(true)`
   - Radar mark (sweep wedge and rings on splash/consent) — `.accessibilityHidden(true)`
   - Sweep wedge, dial arcs, and dial center stack — `.accessibilityHidden(true)`

3. **Dynamic Type support**:
   - Capped all three screens (Splash, Consent, Measuring) at `.dynamicTypeSize(...DynamicTypeSize.accessibility2)`
   - Visually verified at accessibility-extra-large: no clipping, all text fully visible
   - Consent's scrollable layout already handles overflow per spec

4. **Reduce Motion**:
   - PulsingDot already respects `@Environment(\.accessibilityReduceMotion)` — disables pulsing animation when enabled
   - SweepWedgeView already respects `@Environment(\.accessibilityReduceMotion)` — disables sweep rotation when enabled
   - Functional updates (arc-length changes, number updates) remain unaffected

## Implementation Notes

- The combined VoiceOver element binds to a live, polled value via `.accessibilityValue()` and `.updatesFrequently`, avoiding per-tick announcements and reducing spam
- All decorative visual elements (sweep wedge, arcs, glyphs, status dots, radar rings) are hidden from accessibility tree, leaving only functional information (speed numbers, labels, buttons, data counters)
- Dynamic Type cap at accessibility2 is consistent across all screens; consent's ScrollView handles overflow
- Reduce Motion support is built into the animation-layer components (PulsingDot, SweepWedgeView) via environment reads; no additional logic needed in parent views
