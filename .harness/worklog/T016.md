# T016 — Live radio provider (CoreTelephony)

## Worklog

### 2026-07-17

**Completed T016 — Live radio provider (CoreTelephony)**

Implemented the on-device CoreTelephony-backed radio generation provider with full unit test coverage.

**Work done:**

1. **RadioTechnologyMapping.swift** — Created pure mapping function from CTRadioAccessTechnology constants to RadioGeneration enum
   - Maps 5G NR/NRNSA → `.fiveG`
   - Maps LTE → `.lte`
   - Maps 3G technologies (WCDMA, HSDPA, HSUPA, CDMA EV-DO Rev A/B/0, eHRPD) → `.threeG`
   - Maps 2G technologies (GPRS, Edge, CDMA 1x) → `.twoG`
   - Defaults to `.unknown` for nil or unrecognized/future values (forward-compatible)

2. **CoreTelephonyRadioProvider.swift** — Implemented RadioInfoProviding protocol
   - Owns one CTTelephonyNetworkInfo instance
   - Yields current generation immediately on stream creation
   - Observes CTServiceRadioAccessTechnologyDidChange notifications
   - Re-reads and yields new generation on each notification
   - Properly handles concurrency (nonisolated static method, @preconcurrency import ObjectiveC)
   - Cleans up observer on stream termination
   - No entitlements, permissions, or Info.plist keys needed

3. **AppEnvironment.swift** — Wired composition root with device/simulator conditional logic
   - On device builds (`#if !targetEnvironment(simulator)`): uses CoreTelephonyRadioProvider if scenario is `.live`
   - Simulator or explicit `-ScoutScenario` launch argument: keeps SimulatedRadioProvider
   - Sampler and path wiring unchanged

4. **RadioTechnologyMappingTests.swift** — Created comprehensive test suite
   - Tests all 13 CTRadioAccessTechnology constants by name (using actual constants, not hardcoded strings)
   - Tests nil → .unknown
   - Tests unrecognized future value → .unknown
   - All 15 tests passing
