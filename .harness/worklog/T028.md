# T028 — RootView: routing UI, splash overlay, stub screens

## 2026-07-17

Built:

- `Sources/App/RootView.swift` — the new `@main` root: owns `@AppStorage("firstRunConsentGiven")`,
  creates `AppRouter` once (seeded from that AppStorage key + the `ScoutHoldSplash` UserDefaults
  flag) and `SweepSession` once via `AppEnvironment.makeSession()`, injects the session with
  `.environment(session)`. ZStack switches on `router.phase` (splash/consent/measuring stub
  screens) with a `SplashStubView` overlay at `.zIndex(1)` + `.transition(.opacity)` that fades
  out after a 1s `.task` sleep calls `withAnimation(.easeOut(duration: 0.4)) {
  router.splashFinished() }`. Starts the session (`session.start()`) at the grant moment
  (consent-stub "Start sweeping" / paused-stub "Start sweeping") and at splash-finish for a
  returning user with consent already given — never when consent is false. Mirrors
  `router.consentGiven` back into the `@AppStorage` var via `.onChange`.
- `Sources/Screens/SplashStubView.swift`, `ConsentStubView.swift`, `MeasuringStubView.swift` —
  minimal stub screens on `ScoutTheme.background`, full-screen (`.ignoresSafeArea()`), with the
  accessibility identifiers the spec requires (`splash.wordmark`, `consent.start`,
  `consent.notNow`, `measuring.hero`, `paused.title`, `paused.startButton`).
- `Sources/ScoutApp.swift` — `WindowGroup` now shows `RootView()` instead of `ContentView()`.
- Deleted `Sources/ContentView.swift` (T001 placeholder, no longer referenced).
