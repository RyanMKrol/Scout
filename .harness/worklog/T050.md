# T050 — Session data/time auto-stop backstop

**Date:** 2026-07-20

## What was done

Implemented session data and time backstop protection to auto-stop sweeps when either data usage or elapsed time caps are reached:

### SweepSession changes (Sources/Domain/SweepSession.swift)
- Added `backstopReached` observable state to track when caps are crossed
- Added static constants for caps: `backstopDataCapBytes` (500 MB) and `backstopTimeCapDuration` (5 minutes)
- Track session start time via injected clock for time-based cap enforcement
- Added `checkBackstop()` method that fires on sample record if measuring (not paused) and either cap is crossed → calls `stop()` and sets `backstopReached = true`
- Added `resetBackstop()` method to clear caps and prepare for another sweep allowance
- Backstop caps never fire while paused (paused state short-circuits the check)

### UI changes (Sources/Screens/MeasuringView.swift)
- Added `backstopWarning()` overlay that displays when `session.backstopReached` is true
- Extracted helper functions `backstopWarningHeader()` and `backstopWarningButtons()` to respect function length limits
- Warning presents an icon, title ("Data cap reached"), and explanatory message
- Two dismissible actions: "Continue sweeping" (resets counters and resumes), "Stop" (dismisses warning and ends session)
- All warning elements have stable `accessibilityIdentifier`s for testing/automation

### Tests (Tests/SweepSessionTests.swift)
- `testBackstopFiresOnDataCapCrossing`: verifies data cap triggers auto-stop mid-measuring
- `testBackstopFiresOnTimeCapping`: verifies time cap triggers auto-stop using injected clock
- `testBackstopNeverFiresWhilePaused`: confirms backstop is inert during pause
- `testResetBackstopClearsCountersAndAllowsMeasuringToContinue`: verifies reset clears bytes and time, allowing fresh allowance

## Observations

The backstop warning does not appear in a brief test run because the caps are high (500 MB / 5 min). On a real device or over a longer session, the warning will trigger and display when either cap is crossed. The injected clock in tests allows unit tests to verify cap triggering without waiting 5 minutes or transferring 500 MB.

The feature seamlessly integrates with existing pause/resume logic — backstop is silenced during pause, which makes sense since paused state doesn't consume data or time against the caps.
