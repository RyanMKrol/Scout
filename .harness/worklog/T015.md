# T015 — Paused states: No-cellular fallback + consent-declined idle

## Date: 2026-07-17

## Summary

Implemented full-fidelity rendering of the two non-measuring paused variants in `MeasuringView.swift`:

**Variant A — No cellular** (consent given, `!session.cellularAvailable`):
- Consolidated the paused content rendering into a structured layout with spacing matching the measuring view
- Status row: static dot at `white(0.28)` + "PAUSED" label at `white(0.4)`, 14pt semibold, tracking 2.5
- Idle dial rendered with `SweepDialView(mode: .idle)` showing em dash and "Mbps"
- Footer with title "No cellular to measure" (20pt semibold, `white(0.82)`)
- Body text with emphasized "cellular" word (15pt base, semibold `white(0.7)` for emphasized word)
- No button — automatically resumes when `cellularAvailable` flips back

**Variant B — Idle / sweeping off** (consent NOT given):
- Same status row and idle dial
- Footer title: "Sweeping is off" (20pt semibold, `white(0.82)`)
- Body: "Scout only measures cellular data when you choose. Start sweeping to measure this spot." (15pt, `white(0.5)`, centered, max width 280pt)
- Button (id `paused.startButton`): "Start sweeping" — 240pt max width, 50pt height, corner radius 14, fill `great` green, label 17pt semibold `onAccent`
- Button triggers `onStart` callback, same as consent screen's primary button

## Implementation details

- Refactored `statusRow()` to be parameterized (`paused: Bool` flag) so both measuring and paused states use consistent styling with appropriate colors/dots
- Created `pausedFooter()` that branches on `consentGiven` to select between `noCellularFooter()` and `idleFooter()`
- Implemented `bodyWithEmphasis()` helper to build attributed text splitting on a word and applying different styling to emphasized vs normal segments
- Removed old stub `pausedContent()` boilerplate
