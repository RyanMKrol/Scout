# T007 — SweepSession: the observable measurement session engine — Worklog

## 2026-07-17 — Completed

**Task:** Build the `@MainActor @Observable` `SweepSession` state machine the UI observes —
consumes the injected sampler/radio/path streams, publishes capped download/upload Mbps,
quality, generation, per-direction session byte counters, and cellular availability, with the
P0 rule that `stop()` immediately tears down all data flow.

**Implementation:**
- Created `Sources/Domain/SweepSession.swift` with exactly the public shape from the spec.
  `start()` is idempotent and spawns three consuming `Task`s (sample stream, radio stream, path
  stream). Each direction gets its own `ThroughputWindow(window: .seconds(2))`; on sample
  arrival the matching window records the sample, `downloadMbps`/`uploadMbps` are recomputed
  and capped at `ScoutMeter.downloadCapMbps` / `uploadCapMbps` using `sample.endedAt` as `now`
  (no owned clock), `quality` is re-derived from `downloadMbps`, and the matching session byte
  counter accumulates (never resets while the app runs).
- Path stream `false` cancels the sampler-consuming task (tearing down the transfer via
  `AsyncStream` cancellation → `onTermination`), zeroes both Mbps readings (quality follows),
  and keeps the byte counters. Path stream `true` while `isMeasuring` re-subscribes to
  `sampler.samples()` with fresh windows.
- `stop()` synchronously cancels all three consuming tasks and sets `isMeasuring = false`;
  readings keep their last values. Only imports `Foundation` + `Observation`, keeping all
  cellular/radio knowledge behind the injected protocols.
- Added `Tests/SweepSessionTests.swift` with hand-rolled fakes (`FakeThroughputSampler` records
  subscribe/terminate counts and exposes a `yield`, driven by a captured `AsyncStream`
  continuation with `onTermination`; `FakeRadioInfoProvider` / `FakeCellularPathMonitor`
  likewise). Tests assert: 250_000 B / 0.2 s download sample → `downloadMbps == 10.0`,
  `quality == .great`, `sessionDownloadBytes == 250_000`; upload samples land only in
  `uploadMbps` (capped at 5) / `sessionUploadBytes`; byte counters accumulate across multiple
  samples split by direction; `stop()` terminates the fake sampler's stream (P0) and clears
  `isMeasuring`; path `false` drops `cellularAvailable`, zeroes readings, and terminates the
  sampler stream; path `true` again triggers a second `samples()` subscription.
- Test-first: ran the new test file before implementing `SweepSession` — it failed to compile
  (type didn't exist), confirming it exercised real behavior, then implemented until the suite
  went green.
